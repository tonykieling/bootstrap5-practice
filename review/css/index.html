1- CSS Box Model

HTML elements can be considered boxes.

It contains:

  • Margin is a transparent area outside the border. It clears an area after the border.
  • Border goes around the padding and the content.
  • Padding is like the margin for the content.
  • Content yelds the content, text and images.

The full size of an element has to consider not only the content width and height, but also padding and border.

Margin counts to the space taken by an element, not specificaly for its total size.

Outline is a property that draws a border without modifying the size of the element - border does it.

Box Model Properties:

  • Height is relative to the vertical size of an element's content.
  • Width is the horizontal size of that element.
    • they change the size of block elements
  • Margin is the space around the border of an element. It is always transparent.
  • Border is the framework around the content. It can be colored and in diff forms such as solid and pointed.
  • Padding is the space around the content of an element.

    • box-sizing: border-box changes the size of the box to the size of the actual content box (which will shrink it).

    Inline Elements

    They take up the same space as their content. It changes line when it is not possible to fit in that line.

    Examples are <a>, <span> and <strong>

    Block Elements

    They expand to the whole width of their container element, regardles of thei content width. It means that a block element always start a new line.

    Examples are <div>, <h1>, <p> and <section>

    Display Property

    It is used to change de default behavior of inline and block-level elements.

    Display: inline will turn a block element into a inline one.

    Display: inline-block will turn a block element into a inline one being possible to set its size.

    Display: block will turn a inline element into a block one.

    Position vs Grid

    • Static is when the element is positioned according to the normal flow of the document. The top, right, bottom, left, and z-index properties have no effect. This is the default value for any html element.
    • Relative element is positioned according to the normal flow of the document, and then offset relative to itself based on the values of top, right, bottom, and left. The offset does not affect the position of any other elements; thus, the space given for the element in the page layout is the same as if position were static. This value creates a new stacking context when the value of z-index is not auto. Its effect on table-*-group, table-row, table-column, table-cell, and table-caption elements is undefined.

      Relative keeps the flow of the document and it is possible to use z-index, top, right, etc.

      The trbl position is relative to their current/initial position.

    • Absolute element is removed from the normal document flow, and no space is created for the element in the page layout. It is positioned relative to its closest positioned ancestor, if any; otherwise, it is placed relative to the initial containing block (most of the time is body). Its final position is determined by the values of top, right, bottom, and left. This value creates a new stacking context when the value of z-index is not auto. The margins of absolutely positioned boxes do not collapse with other margins.

      Absolute element takes off the flow of the document and it will take the reference of the first parent(s) element which is relative positioned.

      They are gonna be positioned according to its parent element.

    • Fixed element is removed from the normal document flow, and no space is created for the element in the page layout. It is positioned relative to the initial containing block established by the viewport, except when one of its ancestors has a transform, perspective, or filter property set to something other than none (see the CSS Transforms Spec), in which case that ancestor behaves as the containing block. (Note that there are browser inconsistencies with perspective and filter contributing to containing block formation.) Its final position is determined by the values of top, right, bottom, and left. This value always creates a new stacking context. In printed documents, the element is placed in the same position on every page.
    • Sticky element is positioned according to the normal flow of the document, and then offset relative to its nearest scrolling ancestor and containing block (nearest block-level ancestor), including table-related elements, based on the values of top, right, bottom, and left. The offset does not affect the position of any other elements. This value always creates a new stacking context. Note that a sticky element "sticks" to its nearest ancestor that has a "scrolling mechanism" (created when overflow is hidden, scroll, auto, or overlay), even if that ancestor isn't the nearest actually scrolling ancestor. This effectively inhibits any "sticky" behavio

      • All elements can be positioned, unless the static position one.

    Flexbox and Grid

    They both helps to align and distribuite space among items in a container easily.

    Flexbox handles one dimension at at time - row or column.

    Grid works in two dimension, arraging multiple lines and columns at at time.


    Flexbox layout is more appropriate to the components of an application and small-scale layouts
    Grid layout is intended for larger scale layouts.

    Flexbox

    It uses:

    Flex Container, which is the parent element, and

    Flex Items, refering the child elements for a flexbox element.

    As Flexbox elements are arrangeded in one dimension, the default direction is horizontal (X axis), but it can be changed to the vertical one (Y axis).

    flex-direction property does that

    Flexbox concepts and terminology

    Flexbox properties

    Parent - Flex Container

    1. display: flex; /* or inline-flex */ -- sets the element to be a flex one.
      2. Difference between flex and inline-flex:
    2. flex-direction: row | row-reverse | column | column-reverse; defines the directions for its items.
    3. flex-wrap: nowrap | wrap | wrap-reverse; by default all items will try to fit in one line, but this property can change this behaviour.
    4. flex-flow: column wrap; it is a short for flex-direction and flex-wrap.
    5. justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly | start | end | left | right ... + safe | unsafe; it distributes space along the main axis - horizontally.
    6. align-items: stretch | flex-start | flex-end | center | baseline | first baseline | last baseline | start | end | self-start | self-end + ... safe | unsafe; how flex items are laid out along the cross axis on the current line. Think of it as the justify-content version for the cross-axis (perpendicular to the main-axis - vertically).
    7. align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch | start | end | baseline | first baseline | last baseline + ... safe | unsafe; aligns a flex container’s lines within when there is extra space in the cross-axis, similar to how justify-content aligns individual items within the main-axis.
      !!!!! This property only takes effect on multi-line flexible containers, where flex-flow is set to either wrap or wrap-reverse). A single-line flexible container (i.e. where flex-flow is set to its default value, no-wrap) will not reflect align-content.

    Children - Flex Items

    1. order: 2; when applied for each item individually, it changes the order that the element are presented in the elements flow.
    2. flex-grow: 2; /* default 0*/ This defines the ability for a flex item to grow if necessary. It accepts a unitless value that serves as a proportion. It dictates what amount of the available space inside the flex container the item should take up.
      If all items have flex-grow set to 1, the remaining space in the container will be distributed equally to all children. If one of the children has a value of 2, the remaining space would take up twice as much space as the others (or it will try to, at least).
    3. flex-shrink: 3; /* default 1 */ it enables a flex item to shrink, if necessary.
    4. flex-basis: | auto; /* default auto */ defines the default size of an element before the remaining space is distributed. It can be a length (e.g. 20%, 5rem, etc.) or a keyword. The auto keyword means “look at my width or height property”. The content keyword means “size it based on the item’s content” – this keyword isn’t well supported yet, so it’s hard to test and harder to know what its brethren max-content, min-content, and fit-content do.
      If set to 0, the extra space around content isn’t factored in. If set to auto, the extra space is distributed based on its flex-grow value. See below.
    5. flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]; short for flex-grow, flex-shrink, and flex-basis. The default is 0 1 auto
    6. align-self: auto | flex-start | flex-end | center | baseline | stretch; it allows the default alignment (or the one specified by align-items) to be overridden for individual flex items. More info on align-items.

    1- Nesting selectors

    There are 4 types of combinators in CSS:

    1. Descendent selector --> space
    2. Child selector --> >
    3. Adjacent sibling selector --> +
    4. General sibling selector --> ~
    Example 1 - descendent selector: <space>
    Code HTML: 
              <div class="descendent-selector">
            <p>Test - with style</p>
          </div>
          <p>Test - without style</p><div class="descendent-selector">
            <p>Test - with style</p>
          </div>
    CSS: 
              .descendent-selector p {
            margin: 0;
            background-color: blue;
            color: darkred;
          }
     Output:

    Test - with style

    Test - without style

    Test - with style
    Example 2 - child selector: +
    Code HTML: 
              <div>
            <p>This is 'p' inside a 'div'<span> - and this is a 'span' inside that 'p'</span></p>
            <span class="span-temp">And this is a 'span' inside, child, a div.</span>
          </div>
    CSS: 
              div > .span-temp {
            color: gold;
            background-color: indigo;
          }
    Output:

    This is 'p' inside a 'div' - and this is a 'span' inside that 'p'

    And this is a 'span' inside, child, a div.
    Example 3 - adjacent sibling selector: ~
    Code HTML: 
              <div>
            <p>'p' line 1 inside 'div'</p>
            <p>'p' line 2 inside 'div'</p>
          </div>
          <p>'p' line 3 outside, right after, a 'div'</p>
          <p>'p' line 4 outside 'div'</p>
          <div>
            <p>'p' line 5 inside 'div'</p>
          </div>
          <p>'p' line 6 outside, right after, a 'div'</p>
    CSS: 
              div + p {
            background-color: sandybrown;
            text-align: center;
          }
    Output:

    'p' line 1 inside 'div'

    'p' line 2 inside 'div'

    'p' line 3 outside, right after, a 'div'

    'p' line 4 outside 'div'

    'p' line 5 inside 'div'

    'p' line 6 outside, right after, a 'div'
    Example 4 - general sibling selector
    Code HTML: 
              <div>
            It is a 'div'
          </div>
          <div class="m1">this is a 'div' which selector is a class called 'm1'</div>
          <p class="m1">It is a 'p' class = 'm1'</p>
          <h4>this is a plain 'h4'</h4>
          <h3 class="m2">a 'h3' class = 'm2'</h3>
    CSS: 
              div ~ .m1, .m2 {
            text-align: right;
            color: aquamarine;
            background-color: gray;
          }
    Output:
    It is a 'div'
    this is a 'div' which selector is a class called 'm1'

    It is a 'p' class = 'm1'

    this is a plain 'h4'

    a 'h3' class = 'm2'

    Code HTML: 
              <div class="ex1 ex2">
            It is a example that applies 2 classes at the same time.
          </div>
    CSS: 
              .ex1 {
            color: white;
